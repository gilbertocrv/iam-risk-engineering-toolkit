IAM Access Model — RBAC · PAM · SoD

Como os controles de acesso da ISO/IEC 27001:2022 podem ser estruturados de forma operacional. A norma define o que deve ser controlado. Este modelo organiza como esses controles são aplicados na prática.

Premissa

A ISO/IEC 27001:2022 define o que deve ser controlado — acesso, privilégio, segregação. Ela não define o como. RBAC, PAM, SoD, ABAC e PBAC são formas de operacionalizar esses controles em ambientes reais. Esses conceitos não são definidos diretamente pela norma, mas são o que transforma controles abstratos em modelos estruturados, auditáveis e executáveis.

Conceitos do modelo
BASE · CAMADA 1
RBAC
Organiza acesso por função, não por usuário. É a base sem a qual análise de risco não tem estrutura.
5.158.3
RBAC é a base.
PRIORIDADE · CAMADA 2
PAM
Classifica acessos por impacto. Nem todo acesso precisa do mesmo nível de controle.
8.2
PAM prioriza risco.
ANÁLISE · CAMADA 3
SoD
Detecta combinações de acesso que geram risco. Uma role pode ser segura; duas juntas podem criar um vetor crítico.
5.3
SoD é a camada final de análise de risco.
CONTEXTO · COMPLEMENTAR
ABAC
Controla acesso com base em atributos — localização, dispositivo, risco de sessão. Quando RBAC não é suficiente.
5.15
ABAC adiciona contexto ao acesso.
POLÍTICA · COMPLEMENTAR
PBAC
Centraliza decisões de acesso em políticas. Para ambientes complexos onde a lógica de acesso não cabe em roles simples.
5.15
PBAC centraliza a decisão.
Fluxo dentro do ciclo IAM

Cada etapa depende da anterior. RBAC sem PAM deixa privilégios descontrolados. PAM sem SoD não detecta combinações de risco.
