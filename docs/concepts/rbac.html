RBAC Role-Based Access Control

Função

Organizar acesso por função, não por usuário

RBAC define que o acesso deve ser concedido ao papel, não ao indivíduo. No Microsoft Entra ID, isso se traduz em Directory Roles para funções administrativas e Security Groups para funções de negócio.

Problema que resolve

  • Permission creep — acúmulo gradual de acesso
  • Concessão direta por usuário sem estrutura
  • Dificuldade de revisão e auditoria
  • Acesso mantido após mudança de função

Aplicação no Entra ID

  • Directory Roles — funções administrativas nativas
  • Security Groups — funções de negócio
  • Nomear por função: Finance-Admins, não Ana-Acessos
  • Atribuição via PIM para roles privilegiadas

5.158.3

Papel no modelo: RBAC é a base. Sem ele, não existe estrutura para análise de risco. SoD não consegue detectar combinações se não há semântica de função definida. PAM não consegue classificar impacto se todo acesso parece igual.
