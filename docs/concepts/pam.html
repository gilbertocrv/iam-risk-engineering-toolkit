PAM Privileged Access Management

Função

Controlar acessos com maior impacto

PAM identifica contas privilegiadas e aplica controles proporcionais ao risco. Nem todo acesso precisa do mesmo nível de controle. O modelo de tiers é a forma mais prática de estruturar essa classificação.

Problema que resolve

  • Contas privilegiadas sem MFA
  • Falta de rastreabilidade em ações críticas
  • Contas de serviço com roles admin sem vault
  • Acesso privilegiado inativo e esquecido

Aplicação no Entra ID

  • PIM — Privileged Identity Management
  • Tiering — Tier 0 a Tier 2
  • Contas dedicadas para Tier 0
  • Just-in-time access para Tier 1

Modelo de tiers

Tier 0 — controle do tenant (Global Admin, PRA)
Tier 1 — admin de serviços (User, Exchange, Intune)
Tier 2 — acesso elevado (Security Reader, Helpdesk)
Tier 3 — acesso padrão

Controle ISO

8.2 — Privileged Access Rights
Controle da atribuição, uso e revogação de acessos privilegiados. Exige identificação, monitoramento e revisão.

8.2

Papel no modelo: PAM prioriza risco. Define quais identidades exigem controles adicionais — MFA obrigatório, aprovação, monitoramento contínuo. Sem PAM, todas as contas recebem o mesmo tratamento, independentemente do impacto potencial.
