SoD Conflict Analyzer

Detecta combinações de roles do Microsoft Entra ID que introduzem risco de Segregação de Funções. SoD não analisa acessos isolados — analisa combinações. Uma role pode ser segura; duas roles juntas podem criar risco crítico.

Premissa — por que combinações importam

No Microsoft Entra ID, certas combinações de roles criam situações onde um único usuário pode executar ações que deveriam exigir aprovação de outra pessoa — violando o princípio de Segregação de Funções (ISO 5.3). A análise de SoD não questiona se o acesso é necessário; questiona se a combinação é controlada. Exemplo: User Administrator + Authentication Administrator permite que um usuário crie contas e gerencie os métodos de autenticação dessas mesmas contas, sem necessidade de aprovação adicional.
